the idea



tl;dr - by routing data packets through multiple hubs, we offer more safety and reliability than a traditional server system.





For enterprises all around the world, server downtimes are often costly and destructive. Server downtimes negatively affect enterprises and their entire user base. Downtimes can be caused by a variety of factors, such as human error and network outages. A large part of the problem stems from how every client is routed to a singular server, which allows for large scale disasters that affect the entire user base.





To solve this problem, our team devised a decentralized, self-healing networking framework. Our framework alleviates the issue by designating clients to hubs, which act as intermediary transfers. Hubs are clients given the role of a mini-server. When a hub goes down, the clients linked to it will either connect to an existing hub, or form a new hub. To ensure zero data loss, all hubs are backed up with the same information a server would have.

For example, if a new node was introduced into the network, it would ping the server for a list of all hubs (1). The node then will ping all available hubs (2), and if one is available, it will join the closest (least amount of time to ping) (3). If a hub reaches a certain threshold, it will greedily split into two hubs.
